## 簡易

從上篇知道了我們可以在頁面讀取完成後加載`content.js`並拋出資料，`popup.html`就可以即時接收到資料並處理UI。

而目前我考量到的是訂單在多人執行的時候，可能一下子同事A修改商品內容，過陣子同事B又修改訂單地址，因為客人總是有需求，所以我想就簡單一點，在頁面都準備好的時候，我們透過新開/刷新分頁的方式拋出最新的資料，並且在popup頁面中接收資料後顯示出來。

[Google extension 有很多的 API 可以使用](https://developer.chrome.com/docs/extensions/reference/tabs/)，其中一個 `chrome.tabs.update` 是這次我需要的

```js
//
chrome.tabs.query(
  { url: '*://admin.shoplineapp.com/*/orders/*' }, // 我是針對shopline來加載，可以替換任意網址
  function (tabs) {
    for (let i = 0; i < tabs.length; i++) { 
        chrome.tabs.update(tabs[i].id, { url: tabs[i].url }); 
    }
  }
);
```
透過 API 可以抓出目前瀏覽器符合 `url` 的分頁數量，並且用 `for` 迴圈將這些頁籤全部刷新，刷新後可以透過我們已經知道的 `chrome.runtime.onMessage.addListener` 來接收訊息

```js
chrome.runtime.onMessage.addListener(function (
  request,
  sender,
  sendResponse
) {
  // 要做的事情
});
```
